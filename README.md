# SubNexus

## Ссылка доступа: <https://subnexus.streamlit.app/>

**SubNexus** — MVP-сервис для управления подписками в мобильном банке.  
Разработан командой **418:I'm teapot** в рамках чемпионата **«Технологии больших данных и искусственного интеллекта»**.

Приложение автоматически определяет регулярные списания клиентов, показывает активные подписки, предупреждает о ближайших платежах и помогает находить лишние или редко используемые подписки.

Проект написан на Python с использованием **Streamlit** (быстрое создание веб-интерфейса), **Pandas** (обработка данных) и **Faker** (генерация синтетических транзакций). В MVP используются простые эвристики вместо сложного ML, что позволяет быстро развернуть и демонстрировать функционал.

### Описание модулей

- **`data_generator.py`** — генерирует синтетические данные для заданного количества пользователей. Создаёт транзакции за последние N месяцев, включая:
  - подписки (ежемесячные, годовые, недельные) из списка `POPULAR_SERVICES` (реальные цены российских сервисов);
  - случайные шумовые платежи (кафе, такси, супермаркеты) — они не считаются подписками;
  - для каждой подписки с вероятностями `TRIAL_PROBABILITY`, `PRICE_RISE_PROBABILITY`, `UNUSED_PROBABILITY` генерируются пробный период (первый платёж 0), рост цены на 25% через 30 дней, или «забытая» подписка.
  - Функция `generate_all_users_transactions(num_users)` возвращает словарь `{user_id: DataFrame}`.

- **`subscription_detector.py`** — анализирует транзакции одного пользователя и выявляет регулярные списания. Алгоритм:
  1. Игнорирует продавцов из `BLACKLIST_MERCHANTS`.
  2. Проверяет минимальную медианную сумму (`MIN_SUBSCRIPTION_AMOUNT`).
  3. Для каждого продавца вычисляет интервалы между платежами и определяет периодичность (7, 28–31, 365 дней с допуском ±2 дня).
  4. Отсеивает продавцов с недостаточным числом повторений (`MIN_MONTHLY_OCCURRENCES`, `MIN_YEARLY_OCCURRENCES`).
  5. Возвращает DataFrame с полями: название сервиса, медианная сумма, период в днях, даты первого/последнего/следующего платежа.

- **`status_tracker.py`** — присваивает статус каждой найденной подписке на основе текущей даты и истории транзакций. Правила (в порядке приоритета):
  - *Пробный период* — если первая сумма = 0, есть платные позже, и следующий платёж в будущем.
  - *Скоро списание* — если следующий платёж через 1–3 дня.
  - *Цена выросла* — если последняя ненулевая сумма ≥ на 20% выше первой ненулевой (или выше медианы предыдущих).
  - *Возможно не используется* — если последний платёж был более `UNUSED_MONTHLY_DAYS` (30) дней назад (для месячных) или более `UNUSED_YEARLY_DAYS` (180) для годовых.
  - *Активна* — остальные.

- **`analytics.py`** — вспомогательные функции:
  - `total_monthly_cost` — пересчёт всех подписок к месячной стоимости.
  - `top_expensive` — топ N самых дорогих в месяц.
  - `potential_savings` — определяет до 2-х подписок со статусом «Возможно не используется» и сумму их месячной стоимости.
  - `simulate_savings` — возвращает новую общую сумму после гипотетического отключения переданных подписок.

- **`config.py`** — централизованное хранилище всех параметров:
  - пороги детекции (`MIN_MONTHLY_OCCURRENCES`, `MONTHLY_INTERVAL_MIN/MAX` и т.д.);
  - вероятности для генерации специальных кейсов (`TRIAL_PROBABILITY` = 0.7, `PRICE_RISE_PROBABILITY` = 0.7, `UNUSED_PROBABILITY` = 0.5);
  - `POPULAR_SERVICES` — список реальных сервисов с ценами и периодичностью (более 100 позиций);
  - `BLACKLIST_MERCHANTS` — названия, которые никогда не будут подписками;
  - `NUM_MONTHS_HISTORY = 12` (глубина истории транзакций).

- **`app.py`** — главный файл, который запускает Streamlit-приложение. Реализует два режима:
  - *Главная страница* — форма выбора количества клиентов, кнопка генерации, карточки клиентов с пагинацией, поиск по ID, фильтр по проблемам (Лишние / Пробный период / Рост цены). Кнопка «Подробнее» ведёт на детальную страницу.
  - *Детальная страница клиента* — показывает:
    - метрики (общая сумма, количество подписок, потенциальная экономия);
    - топ самых дорогих подписок;
    - форму для **ручного добавления подписки** (название, сумма, периодичность, дата первого платежа);
    - витрину подписок с бейджами статусов;
    - **встроенные рекомендации** для проблемных подписок (с объяснением причины и кнопкой симуляции отключения);
    - **управление подпиской** (иконки: ⭐ важная, 🔔/🔕 уведомления, 👁️‍🗨️ скрыть, ⚠️ ошибочна);
    - **автоматические уведомления** при загрузке страницы для событий: скоро списание, рост цены, пробный период, ежемесячная сводка;
    - тестирование уведомлений по кнопкам (для демонстрации);
    - разворачиваемую таблицу всех транзакций.

## Как работает пользовательский сценарий

1. Пользователь заходит в приложение, видит логотип и краткое описание.
2. Выбирает количество клиентов и нажимает «Сгенерировать данные».
3. Приложение генерирует случайные транзакции для каждого клиента, запускает детектор подписок и трекер статусов.
4. На главной странице отображаются **карточки клиентов** с информацией: ID, число транзакций, число подписок, сумма в месяц, индикаторы проблем (Лишние / Пробный период / Рост цены).
5. При клике на «Подробнее» открывается **детальная страница клиента**, где можно:
   - увидеть метрики и топ дорогих подписок;
   - **добавить подписку вручную** (например, если банк не распознал какой-то платёж);
   - просмотреть список подписок с цветными бейджами статусов;
   - получить **персональные рекомендации** (например, «Вы не пользовались сервисом более 45 дней» или «Стоимость выросла на 25%»);
   - **симулировать отключение** подписки и увидеть новую сумму;
   - отметить подписку как **важную** (исключить из рекомендаций на отключение), **скрыть** (ошибочно определённую) или **отключить уведомления** по ней;
   - протестировать отправку push-уведомлений (кнопки «Скоро списание», «Рост цены», «Пробный период», «Лишняя подписка»);
   - просмотреть все транзакции клиента.
6. Все пользовательские метки (важные, скрытые, отключённые уведомления) сохраняются в `session_state` и не сбрасываются при навигации.  
7. **Автоматические уведомления** показываются один раз за сессию при входе на страницу клиента, не дублируются и учитывают отключённые уведомления по подпискам.

## Запуск проекта локально

### 1. Клонирование репозитория

```bash
git clone https://github.com/AttackBeaver/subNexus
cd subNexus
```

### 2. Создание виртуального окружения

```bash
python -m venv venv
# Windows
venv\Scripts\activate
```

### 3. Установка зависимостей

```bash
pip install -r requirements.txt
```

Содержимое `requirements.txt` (основные пакеты):

```txt
streamlit
pandas
faker
python-dotenv
```

### 4. Запуск приложения

```bash
streamlit run app.py
```

После этого откроется браузер с локальным адресом.

### 5. Настройка параметров

Все параметры изменяются в `config.py`. Например, чтобы увеличить количество клиентов по умолчанию — поменяйте `st.session_state.num_users = 100` в `app.py` (или просто выберите другое значение в UI). Чтобы изменить вероятность появления пробного периода — отредактируйте `TRIAL_PROBABILITY` в `config.py` и перегенерируйте данные.

## Как вносить изменения

1. **Изменить логику генерации данных** — правьте `data_generator.py` (например, добавить новые сервисы в `POPULAR_SERVICES` или изменить процент повышения цены).
2. **Добавить новые статусы** — измените `status_tracker.py` (функция `assign_statuses`).
3. **Улучшить детектор** — `subscription_detector.py` (например, добавить определение подписок с произвольной периодичностью).
4. **Изменить интерфейс** — весь UI в `app.py`. Используются стандартные компоненты Streamlit (`st.columns`, `st.dataframe`, `st.button`, `st.popover` и т.д.).
